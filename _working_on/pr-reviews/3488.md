## Verdict

The fix is correct, minimal, and already reflected in upstream main. A dedicated test (`TestCheckGHPRUsesStateWithoutMergedField`) was added to prevent regression.

---

## Summary

gh CLI v2.89+ removed the `merged` boolean from the `gh pr view --json` surface, making every `gh:pr` gate invocation fail with `Unknown JSON field: "merged"`. This PR removes `Merged bool` from `ghPRStatus`, drops `merged` from the JSON field list, and deletes a now-dead code path in the `CLOSED` handler that used `status.Merged` to distinguish a merged-while-closed PR — a state that is already impossible given that GitHub emits `state=MERGED` in all merge cases.

---

## Strengths

- **Correct root cause fix**: removes the broken field rather than adding a fallback shim.
- **No redundant logic left behind**: the `CLOSED` branch that used `status.Merged` was genuinely dead (GitHub never returns `state=CLOSED` for a merged PR) and its removal simplifies the switch.
- **Doc string updated**: the `gh:pr checks '...'` comment in the help text is kept in sync (`cmd/bd/gate.go:456`).
- **Regression test included**: `TestCheckGHPRUsesStateWithoutMergedField` (gate_test.go:202) installs a fake `gh` binary that asserts `merged` is never requested, then verifies the MERGED path resolves correctly.
- **Security annotation preserved**: `#nosec G204` comment retained on the `exec.Command` call.

---

## Concerns

- **CLOSED test case missing**: `TestCheckGHPRUsesStateWithoutMergedField` only exercises `state=MERGED`. The important regression — that `state=CLOSED` now correctly escalates without the old `if status.Merged` guard — has no dedicated test. The old code could, in theory, have returned `true, false, "was merged"` from the CLOSED branch on a stale/buggy gh version; the new code always escalates on CLOSED, and that invariant deserves a test.
- **`title` field substituted for `merged`**: the diff replaces `--json state,merged,title` with `--json state,title`. `title` was already in the old call, so this is a no-op in terms of fields requested — but the code comment and struct both always had `Title`; the change is clean. No concern here, just confirming it's not a new dependency.
- **No tests on Windows**: the new test skips on Windows due to the POSIX-sh fake-binary approach. This is pre-existing pattern and acceptable given the project's Linux/macOS focus, but worth noting.

---

## Suggestions

- **`cmd/bd/gate_test.go` (near line 240)**: Add a table-driven test or companion to `TestCheckGHPRUsesStateWithoutMergedField` that injects `state=CLOSED` and asserts `resolved=false, escalated=true`. This guards against future regressions in the CLOSED branch.

  // Example addition

  }
